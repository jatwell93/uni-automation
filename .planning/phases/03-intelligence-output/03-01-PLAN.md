   ```python
   SYSTEM_PROMPT = """You are an expert study note generator. Your goal is to create clear, structured study notes from lecture transcripts and slides.
   
   INSTRUCTIONS:
   1. Simplify complex concepts using analogies and plain language
   2. Highlight what students actually need to know
   3. Be concise: avoid padding, keep content high-value
   4. Flag weak areas in the transcript (indicate "⚠️ Low confidence" sections)
   
   OUTPUT FORMAT:
   Generate exactly 6 sections below. Use markdown with clear headers.
   
   ## Summary
   [2-3 sentences: What is this lecture about?]
   
   ## Key Concepts
   [Bullet list: 5-8 core ideas. One sentence each. Order by importance.]
   
   ## Examples
   [3-5 real-world or concrete examples. Show how concepts apply.]
   
   ## Formulas & Key Equations
   [If applicable: definitions, formulas, key numbers. Use LaTeX for math.]
   
   ## Pitfalls & Common Mistakes
   [What students often get wrong. What to watch out for.]
   
   ## Review Questions
   [5-7 self-test questions. Answering these = proof of understanding.]
   
   CONSTRAINTS:
   - Assume reader has no prior knowledge of this topic
   - Flag sections with unclear/missing explanations
   - Total output: 800–1200 words
   - Markdown-valid formatting (valid for Obsidian rendering)
   """
   ```

2. **LLMGenerator class (add methods):**
   - Imports: `from openai import OpenAI, RateLimitError, APIError`
   - Imports: `from tenacity import retry, stop_after_attempt, wait_exponential, retry_if_exception_type`
   - Import logging
   - Constructor: `__init__(config: dict)` - Store OpenRouter API key from config, initialize OpenAI client with base_url="https://openrouter.ai/api/v1"
   - Private method: `_call_llm_with_retry(model: str, messages: list, max_tokens: int = 1500) -> str` - Decorated with @retry (3 attempts, exponential backoff 2-30s, retry on RateLimitError/APIError)
   - Public method: `generate_notes(transcript: str, slide_text: str, model: str = "deepseek/deepseek-chat") -> LLMResult` - Full workflow:
     1. Combine transcript and slide text
     2. Count input tokens (system prompt + combined text)
     3. Validate budget
     4. If over budget: truncate transcript and recalculate tokens
     5. Log pre-flight estimate with token count and cost
     6. Call LLM with retry logic
     7. Return LLMResult with content, tokens, and cost
     8. Handle errors: RateLimitError, APIError, Exception → return LLMResult with error status

3. **Error Handling Details:**
   - RateLimitError (429): Log "Rate limited. Retrying with backoff..." and retry up to 3 times with 2s → 4s → 8s → 30s waits
   - APIError (4xx/5xx): Log error type and message. If 401 (auth), fail immediately with "Invalid OpenRouter API key"
   - Network timeout: Catch and fail with "Network timeout. Check internet connection."
   - Other exceptions: Log full traceback, return error status

4. **Tests in tests/test_llm_generator.py:**
   - test_generate_notes_mock_api() - Mock OpenRouter call, verify return structure (status, content, tokens, cost)
   - test_generate_notes_handles_rate_limit() - Mock 429 error, verify retry happens and eventual success/failure
   - test_generate_notes_handles_auth_error() - Mock 401 error, verify fails immediately without retry
   - test_generate_notes_with_truncation() - Transcript 15000 tokens, verify truncation happens before API call
   - test_generate_notes_cost_calculation_matches_actual() - Verify estimated cost matches OpenRouter response.usage
   - test_generate_notes_system_prompt_included() - Verify system prompt is sent in messages
   - test_generate_notes_respects_budget() - Budget 0.30 AUD, oversized transcript, verify truncation
   - test_generate_notes_returns_markdown() - Verify response.content contains markdown (## headers present)

Implementation notes:

- Log all costs in INFO level for user visibility
