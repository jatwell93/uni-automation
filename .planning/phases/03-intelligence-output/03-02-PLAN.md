  <name>Task 1: Implement markdown validation and frontmatter generation</name>
  <files>src/obsidian_writer.py, src/models.py, tests/test_obsidian_writer.py</files>
  <action>
Create `src/obsidian_writer.py` with markdown formatting capabilities:

1. **MarkdownValidator class:**
   - Method: `is_valid_markdown(content: str) -> tuple[bool, list]` - Check markdown validity. Return (is_valid, [list_of_issues]). Issues detected:
     1. Unmatched code fences (count of ``` must be even)
     2. Unmatched square brackets (count of [ must equal ])
     3. Unmatched curly braces (count of { must equal })
     4. Unmatched parentheses (count of ( must equal ))
     5. Missing required headers (must have ## Summary, ## Key Concepts, ## Examples, ## Formulas, ## Pitfalls, ## Review Questions)
   - Return empty list if all valid
   - Example: (False, ["Unmatched code fence", "Missing ## Formulas section"])

2. **FrontmatterGenerator class:**
   - Method: `generate_frontmatter(lecture_metadata: dict) -> str` - Create YAML frontmatter block. Input dict must contain:
     - `course`: str (e.g., "Business Analytics")
     - `week`: int (e.g., 5)
     - `date`: str (ISO format, e.g., "2026-03-02")
     - `panopto_url`: str (e.g., "https://panopto.com/...")
     - `title` (optional): str for lecture title
   - Output:
     ```yaml
     ---
     course: Business Analytics
     week: 5
     date: 2026-03-02
     tags: [lecture, business-analytics, week-05]
     source: https://panopto.com/...
     ---
     ```
   - Tags auto-generated: ["lecture", "{course_slug}", "week-{week:02d}"] where course_slug = course.lower().replace(' ', '-')

3. **SectionValidator class:**
   - Method: `validate_sections(content: str) -> tuple[bool, dict]` - Parse generated markdown and verify 6 sections exist. Return (all_present, section_details)
   - section_details: {section_name: {present: bool, line_number: int, content_length: int}} for each of ["Summary", "Key Concepts", "Examples", "Formulas", "Pitfalls", "Review Questions"]
   - Example: ("Summary": {"present": True, "line_number": 5, "content_length": 450})

4. **ObsidianNote dataclass (add to src/models.py):**
   - Fields: course: str, week: int, date: str, panopto_url: str, title: str, llm_content: str, frontmatter: str
   - Method: `to_markdown() -> str` - Combine frontmatter + title header + llm_content

5. **Tests in tests/test_obsidian_writer.py:**
   - test_validate_markdown_valid_simple() - Markdown with headers only: returns (True, [])
   - test_validate_markdown_unmatched_code_fence() - "```\ntext" (no closing fence): detects issue
   - test_validate_markdown_unmatched_brackets() - "[text": detects unmatched
   - test_validate_markdown_missing_header() - Content without ## Formulas: detects missing
   - test_validate_markdown_all_sections_present() - Content with all 6 headers: returns (True, [])
   - test_generate_frontmatter_basic() - course="Bio", week=3, date="2026-03-01": generates valid YAML
   - test_generate_frontmatter_tags_generation() - course="Business Analytics": tags include "business-analytics"
   - test_generate_frontmatter_with_title() - Include optional title field: appears in output
   - test_generate_frontmatter_without_panopto_url() - Missing panopto_url: fallback to "N/A"
   - test_validate_sections_all_present() - Content with all 6 section headers: returns (True, section_details)
   - test_validate_sections_missing_one() - Missing ## Pitfalls: detects absence
   - test_validate_sections_line_numbers() - Verify line numbers are correct
   - test_obsidian_note_to_markdown() - ObsidianNote.to_markdown() combines frontmatter + title + content

Implementation notes:
- DO NOT attempt to write files yet (next task)
- Validation must be strict about 6 required sections (LLM-04 requirement)
- Tag generation MUST handle special characters in course names (e.g., "Data & Analytics" → "data-analytics")
- YAML frontmatter must be valid (use simple string formatting, not a YAML library, to avoid extra dependencies)
- All validation methods should be deterministic (same input → same output)

  <name>Task 2: Implement vault file writing with error handling</name>
  <files>src/obsidian_writer.py, tests/test_obsidian_writer.py</files>
  <action>
Add to `src/obsidian_writer.py`:

1. **VaultWriter class:**
   - Import pathlib.Path, datetime, logging
   - Constructor: `__init__(vault_path: str, logger: logging.Logger = None)` - Store vault path, set up logger
   - Method: `verify_vault_exists() -> tuple[bool, str]` - Check if vault_path exists as directory. Return (exists, message). Example: (False, "Vault not found at /path/to/vault. Create folder or update obsidian_vault_path in config.")
   - Method: `write_notes(metadata: dict, content: str) -> tuple[bool, str]` - Full write pipeline:
     1. Validate vault exists (call verify_vault_exists)
     2. Ensure subfolder exists (vault_path / metadata["subfolder"]) using mkdir(parents=True, exist_ok=True)
     3. Generate filename:
        - If week is int: f"Week_{week:02d}.md"
        - Otherwise: f"{date_str}_{course}.md"
     4. Check for conflicts:
        - If file exists: append timestamp to filename (e.g., "Week_05__20260302_143022.md")
        - Log warning: "File exists at {old_path}. Saving as {backup_path}"
     5. Validate markdown before writing (use MarkdownValidator)
     6. Write file with UTF-8 encoding
     7. Return (success, file_path_or_error)
   - Method: `list_notes() -> list[str]` - Return list of .md files in vault (for user reference)

2. **File Writing Details:**
   - Write atomically: write to temp file first, then rename (prevents partial writes if interrupted)
   - Use `pathlib.Path.write_text(full_content, encoding='utf-8')`
   - Log all operations: INFO for success, ERROR for failures
   - Error messages MUST be clear and actionable:
     - "Obsidian vault not found at {vault_path}. Create the folder or update obsidian_vault_path in config."
     - "Cannot create subfolder {subfolder}: {error}. Check folder permissions."
     - "Invalid markdown generated: {issues}. This is a bug—please report with transcript."

3. **ObsidianWriter convenience class (orchestrates both validators and writer):**
   - Constructor: `__init__(config: dict, logger: logging.Logger = None)` - Extract vault_path from config
   - Method: `write_complete_note(metadata: dict, llm_content: str) -> tuple[bool, str]` - One-shot method:
     1. Generate frontmatter
     2. Validate markdown
     3. Write to vault
     4. Return (success, file_path_or_error_message)
   - This is the main entry point for integration with Plan 03-01

4. **Tests in tests/test_obsidian_writer.py:**
   - test_verify_vault_exists_true() - Create temp vault, verify returns (True, "Vault OK")
   - test_verify_vault_exists_false() - Non-existent path, returns (False, error_message)
   - test_write_notes_success() - Valid metadata + content, creates file at correct path
   - test_write_notes_creates_subfolder() - Subfolder doesn't exist, write creates it
   - test_write_notes_prevents_overwrite() - File exists, write creates backup with timestamp
   - test_write_notes_invalid_vault_path() - Vault not found, returns (False, error_message)
   - test_write_notes_cannot_create_subfolder() - Subfolder creation denied (mock permission error), clear error
   - test_write_notes_validates_before_writing() - Invalid markdown, returns (False, "Invalid markdown...")
   - test_write_notes_utf8_encoding() - Unicode content (é, 中文, emojis), writes correctly
   - test_list_notes_empty_vault() - No .md files, returns []
   - test_list_notes_with_files() - 3 .md files, returns all 3 paths
   - test_obsidian_writer_end_to_end() - Create metadata, generate frontmatter, validate, write—all succeed
   - test_obsidian_writer_with_overwrite() - File exists, ObsidianWriter creates backup

Implementation notes:
- Use pathlib.Path exclusively (cross-platform, cleaner API)
- Filename generation MUST handle special characters in course names (replace invalid chars with underscore: <, >, :, ", /, \, |, ?, *)
- Timestamp for backup: datetime.now().strftime("%Y%m%d_%H%M%S") (no colons, Windows-safe)
- Temp file: use file.stem + ".tmp" pattern (e.g., "Week_05.tmp" → write → rename to "Week_05.md")
- Logging MUST use Python logging module (not print), severity levels INFO/ERROR
- Tests use tempfile.TemporaryDirectory for isolated vault testing
- No external APIs or network calls
