<task type="auto">
  <name>task 1: Create cookie loading module with browser JSON parsing</name>
  <files>src/auth.py, src/models.py</files>
  <action>
Create `src/auth.py` module with cookie handling functions.

Implement `load_cookies(cookie_file: str | Path) -> requests.cookies.RequestsCookieJar`:
1. Read JSON file from cookie_file path
2. Parse browser JSON format: array of objects with keys: `name`, `value`, `domain`, `path`, `expires`, etc.
3. Tolerate variations in browser export format (Chrome, Firefox, Edge may have slightly different schema)
4. Convert to `requests.cookies.RequestsCookieJar` for use with requests library
5. Handle errors:
   - FileNotFoundError: "Cookie file not found: {path}\nRun cookie extraction instructions..."
   - json.JSONDecodeError: "Cookie file is not valid JSON: {path}\nCheck file format or re-export from browser"
   - ValidationError (missing required fields): "Cookie file missing required fields (name, value). Check export format."

Example input (browser JSON export):
```json
[
  {
    "name": "PANOPTO_COOKIE",
    "value": "xyz123...",
    "domain": ".panopto.university.edu",
    "path": "/",
    "expires": 1234567890,
    "httpOnly": true,
    "secure": true,
    "sameSite": "None"
  },
  ...
]
```

Return: RequestsCookieJar with all cookies loaded and ready to use with requests library

Implement `AuthResult` data class in src/models.py:
```python
@dataclass
class AuthResult:
    success: bool
    message: str
    session_info: Optional['SessionInfo'] = None
    expires_in_seconds: Optional[int] = None

@dataclass
class SessionInfo:
    user_id: str
    username: str
    expires_at: str
```

Test:
- Valid browser export → cookies loaded correctly
- Missing name/value fields → Clear error message
- Malformed JSON → JSON error with helpful message
- File not found → File not found error with recovery instructions
  </action>

<task type="auto">
  <name>task 2: Implement Panopto session validation with API test call</name>
  <files>src/auth.py</files>
  <action>
Create `validate_session(cookies: RequestsCookieJar, panopto_url: str) -> AuthResult` function.

Implementation:
1. Extract base domain from panopto_url (e.g., "https://panopto.university.edu/Panopto/..." → "https://panopto.university.edu")
2. Make test API call to Panopto to verify session is valid:
   - **Strategy A (preferred):** GET {base_url}/api/v1/user/me
   - **Strategy B (fallback):** HEAD request to {panopto_url} with cookies
   - If Strategy A fails, fall back to Strategy B with minimal information
3. Expected success: HTTP 200 response (not 401, 403, or timeout)
4. On success: Extract user info if available (optional enhancement; basic validation is sufficient)
5. Return AuthResult with success=True, session_info (if available)

Error handling - return AuthResult with success=False and actionable message:
- **401 Unauthorized:** "Cookies expired or invalid. Refresh from browser:\n1. Open {panopto_domain}\n2. DevTools (F12) → Storage → Cookies\n3. Export as JSON\n4. Save to {cookie_file}\n5. Re-run: python run_week.py {config_file}"
- **403 Forbidden:** "Access denied. Check that cookies are from correct institution/account."
- **Timeout (>10 sec):** "Panopto API not responding. Check internet connection and try again."
- **Connection error:** "Cannot reach Panopto. Check URL or network connection.\nURL: {panopto_url}"
- **Other HTTP error:** "Panopto returned {status_code}. Try refreshing cookies and re-running."

Timeout: 10 seconds (API should respond quickly)

Return AuthResult object with:
- success: True/False
- message: Descriptive status or error message
- session_info: User ID, username, expiry (optional; success-case enhancement)
- expires_in_seconds: Time until session expires (optional; informational)

Implementation notes:
- Use requests library (already imported from Plan 1)
- Handle network errors gracefully (ConnectionError, Timeout, etc.)
- Log all validation attempts and results (use logging module from src/logger.py)
- Do NOT fail on Strategy A → Strategy B fallback; that's acceptable for Phase 1
- Test call should be read-only (GET, HEAD); do not modify anything on Panopto
