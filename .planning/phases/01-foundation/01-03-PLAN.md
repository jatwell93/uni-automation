  <name>task 1: Implement streaming video download with error handling and cleanup</name>
  <files>src/downloader.py, src/models.py</files>
  <action>
Create `src/downloader.py` module with video download function.

Implement `download_video(video_url: str, output_path: str | Path, cookies: RequestsCookieJar, timeout: int = 300) -> DownloadResult`:

Download logic:
1. Validate output directory exists; create if missing
2. Make HTTP GET request to video_url with cookies, stream=True, timeout={timeout}
3. Stream download in chunks (chunk_size=8192 bytes = 8KB, standard for HTTP streaming)
4. Write chunks to output_path as binary
5. Track progress (bytes downloaded, optional: percentage if Content-Length available)
6. Return DownloadResult with success, file_size, file_path

Error handling (all errors return DownloadResult with success=False):
- **Network timeout:** "Download timeout. Check internet connection and try again. Partial file deleted."
- **Connection error:** "Cannot reach Panopto. Check URL and network connection."
- **404 Not Found:** "Video not found at URL: {url}. Check Panopto URL in config."
- **403 Forbidden:** "Access denied. Check that cookies are fresh and from correct account."
- **Partial download (file size too small):** Detected later by validator; downloader just detects network failure
- **Disk full:** "Not enough disk space for download. Free up space and retry."
- **Permission denied:** "Cannot write to output directory: {output_path}. Check permissions."

Cleanup:
- If download fails (exception or non-2xx status): Delete partial file using try/finally
- Ensure cleanup happens even if exception raised mid-download
- Log cleanup action: "Cleaning up partial file: {output_path}"

Logging:
- Start: "Downloading video from {url}..."
- Progress (optional): "Downloaded {bytes_downloaded}MB..." (every 10% or every 50MB)
- Success: "✓ Downloaded {output_path} ({file_size_mb}MB)"
- Error: "✗ Download failed: {error_message}"

DownloadResult data class (in src/models.py):
```python
@dataclass
class DownloadResult:
    success: bool
    file_path: Optional[Path] = None
    file_size: Optional[int] = None
    message: str = ""
    error: Optional[str] = None
```

Implementation notes:
- Use requests library (stream=True for memory efficiency)
- Timeout: 300 seconds default (can be overridden; long enough for multi-GB downloads on slow networks)
- Do NOT load entire file into memory (use streaming)
- Do NOT resume partial downloads (Phase 4); always re-download from scratch
- Progress output to console: "Downloaded 150MB / 450MB" (if Content-Length available)

Test:
- Valid video_url → File downloaded successfully
- Network timeout → File deleted, clear error message
- 404 error → Clear error message
- Disk full → Error message with recovery instructions
- Permission denied → Error message with recovery instructions

  <name>task 2: Implement ffprobe video validation</name>

Create `src/validator.py` module with ffprobe-based validation.

Implement `validate_video(video_path: str | Path, min_size_mb: int = 100, min_duration_sec: float = 60) -> ValidationResult`:

Validation logic:
1. Check if ffprobe is installed: `shutil.which('ffprobe')`
   - If not found: Return error with clear message: "FFmpeg not installed.\nDownload from: gyan.dev/ffmpeg/builds/\nAfter install, add to PATH or set FFMPEG_HOME env var"
2. Run ffprobe command to extract metadata:
   ```
   ffprobe -v error -show_entries format=duration,size -of default=noprint_wrappers=1:nokey=1:ch=, <video_path>
   ```
3. Parse output (duration in seconds, size in bytes)
4. Validate:
   - duration > min_duration_sec (default 60 seconds; catch 0-byte or corrupted files)
   - size > min_size_mb (default 100 MB; typical 60-min lecture is 400–600MB; threshold catches gross corruption)
   - file_path exists and is readable
5. Return ValidationResult with success, duration, codec (informational), file_size

Error handling (all errors return ValidationResult with success=False):
- **ffprobe not installed:** Clear install instructions (see above)
- **ffprobe timeout:** "Video validation timeout. File may be very large or corrupted. Try again or validate manually."
- **Duration too short:** "Video duration {duration}s is below minimum {min_duration}s. File may be corrupted."
- **File size too small:** "File size {size}MB is below minimum {min_size}MB. Download may have been interrupted."
- **File not found:** "Video file not found: {video_path}"
- **ffprobe subprocess error:** "ffmpeg error: {stderr}. File may be corrupted or unsupported format."

Logging:
- Start: "Validating video {video_path}..."
- Success: "✓ Video valid: {duration}s, {size}MB"
- Error: "✗ Validation failed: {reason}"

ValidationResult data class (in src/models.py):
```python
@dataclass
class ValidationResult:
    success: bool
    duration_seconds: Optional[float] = None
    file_size_bytes: Optional[int] = None
    codec_name: Optional[str] = None
    message: str = ""
    error: Optional[str] = None
```

Implementation notes:
- Timeout: 30 seconds (ffprobe should be very fast even for large files)
- Extract codec_name for informational logging (helps debug format issues)
- Use subprocess.run() with capture_output=True, timeout=30
- Parse ffprobe output carefully (may have extra whitespace)
- On Windows: ffprobe may be in different location; FFMPEG_HOME env var provides override

Test:
- Valid video file → Validation passes with correct duration/size
- File too small → Clear error message
- Duration too short → Clear error message
- ffprobe not found → Clear install instructions
- ffprobe timeout → Error message
- File doesn't exist → File not found error

  <name>task 3: Implement transcript download (API + fallback)</name>
  <files>src/downloader.py</files>
  <action>
Extend `src/downloader.py` with transcript download function.

Implement `download_transcript(session_id: str, output_path: str | Path, cookies: RequestsCookieJar, panopto_base_url: str) -> TranscriptResult`:

Transcript download logic:
1. Strategy A (API): Try to fetch transcript via Panopto API
   - Endpoint: `{panopto_base_url}/api/v1/sessions/{session_id}/transcript` (or similar, exact endpoint may vary)
   - Method: GET with cookies
   - Accept formats: JSON, VTT, SRT, TXT
   - If successful (200 OK): Parse and save to output_path
   
2. Strategy B (fallback): If API fails or returns 404:
   - Log warning: "Transcript API unavailable; skipping transcript"
   - Return TranscriptResult with success=False, message="Transcript not available"
   - Do NOT fail entire pipeline; transcript is enhancement, not requirement

3. If transcript available (Strategy A succeeds):
   - Save raw transcript file (no cleaning in Phase 1; cleaning is Phase 2)
   - Handle various formats: VTT (with timestamps), SRT, TXT, or JSON
   - Use UTF-8 encoding with BOM handling (utf-8-sig)
   - Return TranscriptResult with success=True, file_size, format

Error handling:
- **API endpoint not found (404):** Log warning, skip transcript gracefully
- **Network timeout:** "Transcript download timeout. Skipping transcript; video available."
- **Permission denied (403):** "Cannot access transcript. Skipping; video available."
- **Disk full:** "Disk full when downloading transcript. Skipping."

Logging:
- Attempt: "Attempting to download transcript..."
- Success: "✓ Downloaded transcript ({format}, {file_size})"
- Skip: "⚠ Transcript not available; proceeding with video only"

- Log all attempts but don't fail
