Create src/error_handler.py with ErrorHandler class managing error categorization and retry logic:

**ErrorCategory enum:**
- RETRYABLE: transient errors (timeout, connection reset, rate limit, temporary API error)
- FATAL: permanent errors (invalid config, auth failure, file not found, invalid format)

**RetryableError and FatalError custom exceptions:**
- Both inherit from Exception
- Include fields: error_message (str), stage_name (str), recovery_action (str)

**ErrorHandler class:**
- Static method: categorize(exception: Exception, stage_name: str) → ErrorCategory
  - Checks exception type/message pattern to determine if retryable or fatal
  - Network errors (timeout, connection, DNS): RETRYABLE
  - Auth errors (401, invalid cookie, token expired): FATAL
  - Config errors (missing field, invalid value): FATAL
  - File errors (file not found): FATAL
  - Quota/rate limit errors (429, quota exceeded): RETRYABLE
  - API errors (5xx, service unavailable): RETRYABLE (usually transient)
  - Default unknown errors: RETRYABLE (assume transient, let max_retries control)

- Static method: exponential_backoff(attempt: int, base_delay: float = 2.0, max_delay: float = 30.0) → float
  - Returns delay for attempt N: base_delay * (2^N) + random jitter
  - Attempt 1: 2-3s, Attempt 2: 4-7s, Attempt 3: 8-30s (capped)
  - Example: exponential_backoff(0) → 2.5s, exponential_backoff(1) → 5.3s

- Static method: handle_error(exception: Exception, stage_name: str, max_retries: int = 3) → (should_retry: bool, delay_seconds: float)
  - Returns (True, delay_sec) if should retry, (False, 0) if fatal
  - For retryable errors: return (True, exponential_backoff(attempt_num)) if attempt < max_retries, else (False, 0)
  - For fatal errors: return (False, 0)

**Error message formatting:**
- Retryable: "{error_type}: {message}. Attempt {N}/{max_retries}. Retrying in {delay}s..."
- Fatal: "{error_type}: {message}. Fix required: {recovery_action}"
- Recovery actions:
  - Auth: "Refresh cookies from browser, update config, and retry"
  - Config: "Check YAML syntax and required fields, fix error, and retry"
  - Network timeout: "Check internet connection, retry stage, or increase timeout"
  - API quota: "Check OpenRouter quota, wait and retry, or switch model"
  - File not found: "Check file path in config, ensure file exists, and retry"

**Logging:**
- Each error logged with: timestamp, stage_name, error_type, message, recovery_action
- Format: "[2026-03-02 10:30:45] ERROR ({stage_name}) {error_type}: {message} | Recovery: {recovery_action}"

- Around each stage (download, transcript, audio, slides, llm, output), wrap in try/except with retry logic:

```python
def run_stage(stage_func, stage_name, config, *args, **kwargs):
    max_retries = 3
    for attempt in range(max_retries):
        try:
            result = stage_func(config, *args, **kwargs)
            logger.info(f"✓ {stage_name} completed")
            return result
        except Exception as e:
            should_retry, delay_sec = ErrorHandler.handle_error(e, stage_name, max_retries)
            if should_retry:
                logger.warning(f"Attempt {attempt+1}/{max_retries} failed, retrying in {delay_sec}s", 
                              recovery_action=f"Retrying {stage_name}")
                time.sleep(delay_sec)
            else:
                logger.error(f"Fatal error in {stage_name}: {str(e)}", 
                            recovery_action=ErrorHandler.get_recovery_action(e),
                            exception=e)
                raise
    # Should not reach here
    raise Exception(f"Max retries exceeded for {stage_name}")
```
