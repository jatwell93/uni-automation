<objective>
Implement Google Drive local sync that copies processed artifacts (transcript, audio, slides) to a locally-mounted Google Drive folder, organized by course/week, with validation and quota error handling.

Purpose: After processing a lecture, system should backup cleaned transcript, extracted audio, and slide text to Google Drive for cloud storage and cross-device access. Files organized by course/week (e.g., Business Analytics/Week_05/) for easy navigation. If Google Drive quota full, show clear error and fallback instructions instead of failing silently.

Output: GoogleDriveSyncManager class with copy/validate operations, integrated into pipeline, with config fields for Google Drive path
</objective>

Create src/gdrive_sync.py with GoogleDriveSyncManager class for Google Drive sync:

**GoogleDriveSyncManager class:**

- __init__(config: ConfigModel):
  - Store gdrive_sync_folder path from config
  - Verify folder exists and is writable
  - Raise error if folder missing: "Google Drive sync folder not found: {path}. Set up Google Drive local sync or update gdrive_sync_folder in config."

- sync_artifacts(lecture_id: str, transcript_path: str, audio_path: str, slides_text_path: str, course_name: str, week_number: int) → SyncResult:
  - Create course/week subfolder structure: {gdrive_sync_folder}/{course_slug}/Week_{week_number}/
  - Example: /Google Drive/Business Analytics/Week_05/
  - Course slug: lowercase, spaces→hyphens (e.g., "Data & Analytics" → "data-analytics")
  
  - For each artifact (transcript, audio, slides):
    1. Check if source file exists, return error if not
    2. Copy to target folder: {course}/Week_{week}/artifact_name
    3. Validate copy: check target file exists and size matches source
    4. Log: "✓ Synced transcript ({size} bytes) to {target_path}"
    5. If copy fails (permission, disk full, quota): catch exception, return SyncResult with error and recovery action
  
  - Return SyncResult object with:
    ```python
    @dataclass
    class SyncResult:
        success: bool
        synced_files: int (count of successfully synced files)
        failed_files: int
        total_size_bytes: int
        errors: List[dict] with {file, error, recovery_action}
        sync_duration_seconds: float
    ```

- validate_file_copy(source_path: str, target_path: str) → (success: bool, error_msg: str):
  - Check target file exists
  - Check target file size == source file size (bytes)
  - Return (True, "") if valid
  - Return (False, error_msg) if invalid
  - Error messages:
    - "Target file not found after copy"
    - "Target file size {actual} != source {expected}"
    - "Target file not readable"

- get_course_subfolder_path(course_name: str, week_number: int) → str:
  - Returns: {gdrive_sync_folder}/{course_slug}/Week_{week_number}/
  - Creates folder if missing (mkdir parents=True)
  - Handles special characters in course name (slugify)

**Error handling:**
- File not found: "Source file not found: {path}. Check pipeline output."
- Permission denied: "Cannot write to Google Drive folder. Check permissions or reconfigure path."
- Disk full: "Google Drive quota exceeded or disk full. Free space or increase quota. Retry or use manual backup."
- Invalid path: "Google Drive folder path invalid: {path}. Update config.gdrive_sync_folder."
- Partial copy (size mismatch): "File copy incomplete or corrupted. Source {source_size} != Target {target_size}. Retry sync."

**Logging:**
- Sync start: "Syncing artifacts to Google Drive: {course}/Week_{week}"
- Per file: "✓ Synced {filename} ({size_mb:.2f} MB) → {target_path}"
- Sync complete: "✓ Google Drive sync complete: {N} files, {total_mb:.2f} MB"
- Errors: "✗ Sync failed: {error}. Recovery: {recovery_action}"

**Add to ConfigModel:**
- gdrive_sync_folder: Optional[str] = None
  - Example: "/Users/student/Google Drive/My Drive" (macOS) or "C:\\Users\\student\\Google Drive\\My Drive" (Windows) or "/mnt/google_drive" (Linux)
  - If provided: validate folder exists and is writable (isdir() and access(W_OK))
  - If None: sync skipped (graceful degradation)

- gdrive_sync_enabled: bool = False (default disabled, user must opt-in)
  - If True, require gdrive_sync_folder to be set and valid
  - If False, skip sync (no error)

**Validation logic:**
- If gdrive_sync_enabled=True and gdrive_sync_folder=None: raise ValidationError "Google Drive sync enabled but folder path not set. Set gdrive_sync_folder or disable with gdrive_sync_enabled=false"
- If gdrive_sync_enabled=True and folder doesn't exist: raise ValidationError "Google Drive sync folder not found: {path}. Verify path or disable sync."
- If gdrive_sync_enabled=True and folder not writable: raise ValidationError "Google Drive folder not writable (permissions issue). Check permissions or update path."
