<objective>
Implement checkpoint/resume system that enables failed runs to resume from last successful stage without re-downloading or re-processing completed work.

Purpose: When processing a 50-minute lecture, if LLM call fails at minute 47, system should not re-download video (20 min), re-extract audio (5 min), re-process slides (10 min). Instead, skip to LLM stage and retry. This saves time and reduces API quota waste.

Output: CheckpointManager class (save/load/validate), PipelineState tracking object, and integration with run_week.py --retry flag
</objective>

Create src/checkpoint.py with CheckpointManager class handling checkpoint persistence:

**CheckpointManager class:**
- save(stage_name: str, metadata: dict) → saves JSON checkpoint to .state/{lecture_id}_{timestamp}.json with structure:
  ```json
  {
    "lecture_id": "week_05",
    "timestamp": "2026-03-02T10:30:00Z",
    "stages": {
      "download": {"completed": true, "duration_seconds": 120, "file_size_bytes": 150000000},
      "audio": {"completed": true, "duration_seconds": 45, "file_size_bytes": 5000000}
    },
    "last_completed_stage": "audio",
    "next_stage": "slides"
  }
  ```
- load(state_file: str) → reads JSON, validates structure, returns dict
- validate(checkpoint: dict) → checks JSON format, required fields, stage order (download→transcript→audio→slides→llm→output)
- get_last_completed_stage(checkpoint: dict) → returns stage name or None
- should_skip_stage(checkpoint: dict, stage_name: str) → returns True if stage already completed

**Data structures:**
- PipelineCheckpoint dataclass with fields: lecture_id, timestamp, stages (dict), last_completed_stage, next_stage
- StageStatus enum: PENDING, IN_PROGRESS, COMPLETED, FAILED

**Error handling:**
- FileNotFoundError if state file missing → return None (first run)
- JSONDecodeError if malformed → raise with message "Invalid checkpoint file: {file}. Delete .state/{file} and retry"
- Invalid stage order → raise ValidationError with recovery: "Checkpoint contains invalid stage order. Delete and retry."

**Logging:**
- Log checkpoint save: "✓ Stage {stage_name} completed, checkpoint saved ({file_size} bytes)"
- Log checkpoint load: "↻ Resuming from checkpoint: last completed stage = {stage_name}"

<task type="auto">
  <name>Task 2: Implement PipelineState with resume logic</name>
  <files>src/state.py, tests/test_state.py</files>
  <action>
Create src/state.py with PipelineState class managing pipeline execution flow:

**PipelineState class:**
- __init__(config: ConfigModel, checkpoint_file: Optional[str] = None)
  - If checkpoint_file provided: load checkpoint, determine next_stage, set skip_stages (list of completed stages)
  - If no checkpoint: next_stage = "download", skip_stages = []
  - Store config, checkpoint, next_stage, skip_stages for use by orchestrator

- get_skip_stages() → returns list of stage names to skip (e.g., ["download", "transcript", "audio"])
- get_next_stage() → returns next incomplete stage name (e.g., "slides")
- should_run_stage(stage_name: str) → returns True unless stage in skip_stages
- mark_stage_complete(stage_name: str) → updates internal checkpoint
- cleanup_partial_files(stage_name: str) → removes files from failed stage before retry

**Cleanup logic (cleanup_partial_files):**
- download: delete {lecture_id}.mp4 if file incomplete (size check vs expected)
- transcript: delete {lecture_id}_transcript.* if empty or malformed
- audio: delete {lecture_id}_audio.* if size < 1MB
- slides: delete {lecture_id}_slides_text.* if empty
- llm: delete {lecture_id}_notes.* (can regenerate from checkpoint)
- output: delete Obsidian file (can regenerate from checkpoint)
- Only delete if stage is being retried; do not delete if skipping completed stage

**Error handling:**
- Invalid checkpoint_file path → raise FileNotFoundError with message "Checkpoint file not found: {path}. Run without --retry to start fresh."
- Corrupted checkpoint → raise JSONDecodeError with recovery: "Checkpoint corrupted. Delete .state/{file} and retry."
- Circular dependencies (e.g., slide cleanup before download) → raise ValidationError

**Logging:**
- "Resuming from checkpoint: skipping {len(skip_stages)} completed stages (download, transcript, audio)"
- "Cleaning up partial files from failed {stage_name} stage before retry"
