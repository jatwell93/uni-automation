  <name>Task 1: Implement PIIDetector with pattern matching and optional removal</name>
  <files>src/transcript_processor.py, tests/test_privacy.py</files>
  <action>
Enhance src/transcript_processor.py (from Phase 2) with PIIDetector class:

**PIIDetector class:**
- detect_pii(text: str) → dict with detected PII:
  ```python
  {
    "emails": [list of found emails],
    "names": [list of detected student names],
    "student_ids": [list of patterns like "S12345" or "12345678"],
    "phone_numbers": [list of phone numbers],
    "total_pii_found": int
  }
  ```
  
  Patterns to detect:
  - **Emails:** regex `\b[A-Za-z0-9._%+-]+@[A-Za-z0-9.-]+\.[A-Z|a-z]{2,}\b`
  - **Student names:** list of common first names + common last names (1000+ names) or match patterns like "StudentName XYZ" (all caps)
  - **Student IDs:** regex `\b[Ss]?\d{7,8}\b` (7-8 digits, optional S prefix)
  - **Phone numbers:** regex `\b\d{3}[-.]?\d{3}[-.]?\d{4}\b` (US format) or international `\+\d{1,3}\s?\d{9,}` (simple)
  
  Return empty lists if no matches. Be conservative to avoid false positives (only high-confidence matches).

- remove_pii(text: str, categories: List[str] = None) → str with PII removed:
  - Default categories: ["emails", "student_ids", "names"] (not phone numbers, too risky for partial matching)
  - Replace detected items with [REDACTED] placeholder
  - Example: "John Smith emailed me at john@university.edu" → "[REDACTED] emailed me at [REDACTED]"
  
- log_pii_findings(pii_dict: dict, logger) → logs detected PII to console and error file:
  - Format: "PII Detected: {N} emails, {N} student IDs, {N} names"
  - If config.remove_pii_from_transcript: "Removing PII before LLM call..."
  - If not: "Warning: PII detected in transcript. Set remove_pii_from_transcript=true to remove before LLM call"

**Data structures:**
- PIIResult dataclass: fields total_found, emails_count, names_count, student_ids_count, phone_numbers_count

**Error handling:**
- Invalid regex → log warning, skip pattern, continue
- Empty text → return empty PII dict
- Very large text (>1M chars) → log warning about performance, continue anyway

**Logging:**
- "PII Detection: Found {N} emails, {N} student IDs, {N} names in transcript"
- If remove_pii=true: "✓ PII removed from transcript before LLM call"
- If remove_pii=false and PII found: "⚠ PII detected in transcript. Set remove_pii_from_transcript=true in config to remove."

**Update ConfigModel (src/config.py):**
- Add field: remove_pii_from_transcript: bool = True (default enabled)
- Add validation: check boolean type

<task type="auto">
  <name>Task 2: Implement TempFileManager for tracking and cleanup</name>
  <files>src/temp_manager.py, tests/test_privacy.py</files>
  <action>
Create src/temp_manager.py with TempFileManager class tracking temporary artifacts:

**TempFileManager class:**
- Singleton pattern: TempFileManager.instance() returns shared instance
- register_temp_file(file_path: str, stage: str, description: str = "") → tracks file for cleanup
  - Example: temp_manager.register_temp_file("lectures/week_05.mp4", "download", "Raw video")
  - Stores in dict: {file_path: {stage, description, created_at}}

- register_temp_directory(dir_path: str, stage: str) → tracks directory for cleanup
  - Example: temp_manager.register_temp_directory("/tmp/slide_images_week_05", "slides", "Temporary images for OCR")

- cleanup_all() → removes all registered temp files and directories:
  - For each file: delete if exists, log "[cleanup] Deleted {file_path}"
  - For each directory: delete recursively if exists, log "[cleanup] Deleted directory {dir_path}"
  - Handle errors gracefully: if file already deleted, log and continue
  - Return dict: {deleted_count, failed_count, errors}

- cleanup_by_stage(stage_name: str) → removes only files from specific stage
  - Example: cleanup_by_stage("download") removes only downloaded video, not audio/slides
  - Used for failed stage cleanup (from state.py)

- get_temp_files() → returns list of registered temp files for inspection

- clear_registry() → resets instance (for testing)

**Error handling:**
- File already deleted → log warning, continue cleanup
- Permission denied → log error, continue cleanup
- Invalid path → skip, continue cleanup
- Return summary of deleted count and failures

**Logging:**
- Register: "[temp] Registered temp file: {file_path} ({stage})"
- Cleanup: "[cleanup] Deleted {count} temp files and {count} directories"
- Failed: "[cleanup] Failed to delete {failed_count} files (check permissions): {list}"

**Integration points:**
- In pipeline.py: temp_manager = TempFileManager.instance() at start
- After download: temp_manager.register_temp_file(video_path, "download")
- After audio extraction: temp_manager.register_temp_file(audio_path, "audio")
- After slide extraction: temp_manager.register_temp_directory(slide_images_dir, "slides")
- At end of pipeline: temp_manager.cleanup_all() (in finally block)

- After llm_call(): register generated LLM output as temp (can be regenerated from transcript)

**Cleanup (at end of pipeline, in finally block):**
```python
try:
    # Run pipeline stages...
finally:
    # Cleanup temp files
    temp_manager = TempFileManager.instance()
    cleanup_summary = temp_manager.cleanup_all()
    logger.info(f"✓ Cleanup complete: {cleanup_summary['deleted_count']} files removed")
    if cleanup_summary['failed_count'] > 0:
        logger.warning(f"Failed to delete {cleanup_summary['failed_count']} files (check permissions)", 
                      recovery_action="Manually delete temp files or fix permissions")
```

**Integration with checkpoint/state:**
- If --retry flag used: do NOT re-register files from skipped stages (state.get_skip_stages())
