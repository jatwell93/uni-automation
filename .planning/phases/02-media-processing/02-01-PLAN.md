Main function signature:
```python
def extract_audio(video_path: Path, output_path: Path, codec: str = "aac") -> AudioExtractionResult:
    """
    Extract audio from video file using ffmpeg.
    
    Args:
        video_path: Path to input video file (must exist, must be readable)
        output_path: Path to output audio file (parent dir must exist and be writable)
        codec: Audio codec (default "aac" for MP4 container)
    
    Returns:
        AudioExtractionResult with status, output_path, duration, file_size
    
    Raises:
        AudioExtractionError: If extraction fails (codec unsupported, ffmpeg not found, etc.)
    """
```

Implementation details:
1. **Input validation (before ffmpeg call):**
   - Verify input_path exists: `FileNotFoundError` if missing
   - Verify output directory exists and is writable: Create with mkdir(parents=True) if needed
   - Check input file size > 0: Reject empty videos
   - Use `ffprobe` (subprocess) to validate input video has audio stream:
     - Command: `ffprobe -v error -select_streams a:0 -show_entries stream=duration -of csv=p=0 {video_path}`
     - If no audio stream found: raise `AudioExtractionError("Video has no audio stream")`

2. **FFmpeg extraction command:**
   - Use `typed-ffmpeg` library (modern, type-safe):
     ```python
     from typed_ffmpeg import FFmpeg
     
     ff = FFmpeg()
     ff.input(str(video_path))
     ff.output(str(output_path), acodec="aac", q="5")
     ff.run()
     ```
   - Timeout: 300 seconds (prevent hangs on slow systems)
   - On timeout: raise `AudioExtractionError("Audio extraction timed out (>300s)")`
   - On ffmpeg error: Capture stderr, parse error message, re-raise with context

3. **Output validation (after ffmpeg call):**
   - Call `validate_audio_output(output_path, video_duration)` (see task 2)
   - If validation fails, delete output file and raise error

4. **Error messages (clear, actionable):**
   - "ffmpeg not found. Install ffmpeg from https://www.gyan.dev/ffmpeg/builds/ and add to PATH"
   - "Video has no audio stream. Check Panopto URL and re-download."
   - "Audio extraction timed out (>300s). Video may be corrupted. Re-download and retry."
   - "Audio codec unsupported. Install ffmpeg with full codec support."
   - "Extracted audio too short (10s < 300s expected). Video may be partial. Re-download and retry."

5. **Add to src/__init__.py:** Export `extract_audio`, `AudioExtractionError`, `AudioExtractionResult`

Dependencies to add to requirements:
- `typed-ffmpeg>=3.11` (modern FFmpeg wrapper)
- Already have: requests, pydantic, pyyaml, pathlib (stdlib)

Add validation function to `src/audio_extractor.py`:

```python
def validate_audio_output(output_path: Path, expected_duration: float) -> AudioExtractionResult:
    """
    Validate extracted audio file for integrity.
    
    Checks:
    - File exists and is readable
    - File size ≥ 1MB (non-empty)
    - Duration ≥ 80% of expected (accounts for subtitle-only segments)
    - Audio stream exists (ffprobe check)
    
    Args:
        output_path: Path to extracted audio file
        expected_duration: Duration of original video (seconds)
    
    Returns:
        AudioExtractionResult with validation status
    
    Raises:
        AudioExtractionError: If any validation fails
    """
```

Implementation:
1. **File existence check:**
   - If not exists: raise `AudioExtractionError("Output file not created")`

2. **File size check:**
   - Get file size with `output_path.stat().st_size`
   - If < 1MB (1_048_576 bytes): raise `AudioExtractionError("Extracted audio too small (<1MB). Extraction likely failed.")`

3. **Duration check with ffprobe:**
   - Command: `ffprobe -v error -select_streams a:0 -show_entries stream=duration -of csv=p=0 {output_path}`
   - Parse duration from output (float, seconds)
   - Calculate minimum acceptable: `min_duration = expected_duration * 0.8`
   - If extracted duration < min_duration: raise `AudioExtractionError(f"Audio too short ({actual}s < {min}s expected). Extraction likely failed.")`

4. **Return success:**
   - Create `AudioExtractionResult(status="success", output_path=output_path, duration=actual_duration, file_size=file_size)`

Update `models.py` with:
```python
@dataclass
class AudioExtractionResult:
    status: str  # "success" or "error"
    output_path: Path = None
    duration: float = None
    file_size: int = None
    error_message: str = None
```

Error handling:
- ffprobe failures: Raise `AudioExtractionError("Could not validate audio file. Check ffmpeg installation.")`
- All errors should include recovery instructions in message
