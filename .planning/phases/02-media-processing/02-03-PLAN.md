<objective>
Create the slide text extraction module supporting both text-based PDFs (pdfplumber primary) and image-based PDFs (EasyOCR fallback).

Purpose: Phase 1 provides downloaded PDF slides; Phase 2 must extract text for LLM consumption. Slides may be:
- Text-based: Machine-generated PDFs (pdfplumber fast and accurate)
- Image-based: Scanned documents (require OCR; EasyOCR modern, supports 80+ languages)

This plan implements hybrid approach: try pdfplumber first (fast), fall back to OCR if needed.

Output: Working `SlideExtractor` class with methods for text extraction, image detection, page organization, and graceful handling of unreadable slides.

```python
class SlideExtractor:
    """Extract text from PDF slides (text-based and image-based)"""
    
    def __init__(self):
        self.reader = None  # EasyOCR reader, lazy-loaded if needed
    
    def extract_slide_text(self, pdf_path: Path, use_ocr_fallback: bool = True) -> SlideExtractionResult:
        """
        Extract text from PDF slide file.
        
        Strategy:
        1. Try pdfplumber (fast, good for text-based PDFs)
        2. If < 50% of pages yield text, try OCR fallback (for image-based PDFs)
        3. Combine results: [page 1 text], [page 2 text], ...
        
        Args:
            pdf_path: Path to PDF file
            use_ocr_fallback: If True, use EasyOCR for image-based slides
        
        Returns:
            SlideExtractionResult with organized page text
        
        Raises:
            SlideExtractionError: If file not found or unreadable
        """
    
    def detect_image_slides(self, pdf_path: Path) -> bool:
        """
        Detect if PDF contains mostly image-based pages (scanned).
        
        Returns True if > 50% of pages have no extractable text (indicates scans).
        """
    
    def extract_text_pdfplumber(self, pdf_path: Path) -> dict:
        """
        Extract text from text-based PDF using pdfplumber.
        
        Returns: {page_num: text_content} dict
        """
    
    def extract_text_ocr(self, pdf_path: Path) -> dict:
        """
        Extract text from image-based PDF using EasyOCR.
        
        Returns: {page_num: text_content} dict
        """
```

**extract_text_pdfplumber() implementation:**

Using pdfplumber library (Python-only, no system dependencies):

1. Open PDF: `pdf = pdfplumber.open(pdf_path)`
2. For each page:
   - Call `page.extract_text()` (returns None if page is image-based)
   - Store as `{page_number: extracted_text}`
3. Return dict with page texts
4. Handle errors:
   - FileNotFoundError: "PDF file not found: {path}"
   - pdfplumber.PDFTypeError: "Cannot read PDF. File may be corrupted or unsupported format."
   - Empty result: `all(text is None for text in results.values())` → return empty dict (triggers OCR)

**detect_image_slides() implementation:**

1. Open PDF with pdfplumber
2. Extract text from each page using extract_text()
3. Count pages with no text (None or empty string)
4. Calculate ratio: `image_pages / total_pages`
5. Return `True` if ratio > 0.5 (majority of pages are images)

**extract_slide_text() orchestration:**

```python
def extract_slide_text(self, pdf_path: Path, use_ocr_fallback: bool = True) -> SlideExtractionResult:
    # Validate file exists
    if not pdf_path.exists():
        raise SlideExtractionError(f"Slide file not found: {pdf_path}")
    
    # Try pdfplumber first
    pdfplumber_results = self.extract_text_pdfplumber(pdf_path)
    
    # Check if we got meaningful text
    text_pages = sum(1 for text in pdfplumber_results.values() if text and len(text.strip()) > 10)
    total_pages = len(pdfplumber_results)
    
    # If < 50% pages have text and OCR enabled, try OCR
    if use_ocr_fallback and (text_pages / total_pages if total_pages > 0 else 0) < 0.5:
        print(f"[INFO] Mostly image-based slides detected. Using OCR...")
        ocr_results = self.extract_text_ocr(pdf_path)
        # Merge results: prefer pdfplumber text, fall back to OCR
        final_results = {page: (pdfplumber_results.get(page) or ocr_results.get(page, "")) 
                        for page in pdfplumber_results}
    else:
        final_results = pdfplumber_results
    
    # Organize by page and return
    organized_text = "\n\n".join([f"[Page {i}]\n{text}" for i, text in enumerate(final_results.values(), 1)])
    
    return SlideExtractionResult(
        status="success",
        slide_text=organized_text,
        page_count=total_pages,
        text_pages=text_pages,
        used_ocr=use_ocr_fallback and (text_pages / total_pages < 0.5) if total_pages > 0 else False
    )
```

**extract_text_ocr() implementation:**

Using EasyOCR library (modern, supports 80+ languages, handles poor-quality images):

1. Lazy-load EasyOCR reader on first call:
   ```python
   if self.reader is None:
       import easyocr
       print("[INFO] Loading EasyOCR (first run, ~100MB download)...")
       self.reader = easyocr.Reader(['en'])  # English by default; can extend to ['en', 'other']
   ```

2. Convert PDF pages to images (using Pillow + pdfrw or PIL-based library):
   - For each page in PDF:
     - Convert page to image (PIL Image object)
     - Call `self.reader.readtext(image)`
     - Extract text from results: `[result[1] for result in results]` (result[1] is text string)
     - Join into single page text

3. Return organized dict: `{page_num: ocr_text}`

4. Handle timeouts and errors:
   - OCR timeout (large PDF): Set timeout ~10s per page
   - Low confidence results: Flag with `[OCR uncertain]` prefix
   - Partial failures: Return partial result with error message

**Challenge: PDF → Image conversion**

Options:
- **Option A:** Use `pdf2image` library (requires `poppler` system dependency - complex on Windows)
- **Option B:** Use PyMuPDF/fitz (fast, includes PDF → image conversion, one dependency)
- **Option C:** Use EasyOCR's native support with PDF input (check if available)

**Recommendation:** Use PyMuPDF (`pymupdf` package):
- Single Python package, no system dependencies needed
- Fast PDF → image conversion
- Already used in image processing workflows
- Add to requirements: `pymupdf>=1.23.0`

```python
def extract_text_ocr(self, pdf_path: Path) -> dict:
    import easyocr
    import fitz  # PyMuPDF
    
    # Lazy-load reader
    if self.reader is None:
        self.reader = easyocr.Reader(['en'])
    
    results = {}
    doc = fitz.open(pdf_path)
    
    for page_num, page in enumerate(doc, 1):
        # Convert PDF page to image
        pix = page.get_pixmap(matrix=fitz.Matrix(2, 2))  # 2x zoom for clarity
        image_bytes = pix.tobytes("ppm")
        
        # OCR
        ocr_results = self.reader.readtext(image_bytes)
        
        # Extract text (filter by confidence > 0.3)
        text_lines = [result[1] for result in ocr_results if result[2] > 0.3]
        results[page_num] = "\n".join(text_lines)
    
    doc.close()
    return results
```

**Error handling:**
- EasyOCR not installed: "EasyOCR not installed. Install with: pip install easyocr"
- PyMuPDF import error: "PDF → Image conversion failed. Install: pip install pymupdf"
- OCR timeout: Return partial result with warning "OCR timeout on page {N}. Text may be incomplete."
- Low OCR quality: Flag results with confidence score `[Low confidence]` prefix

**Add dependencies:**
- `easyocr>=1.7.0` (OCR engine)
- `pymupdf>=1.23.0` (PDF → image conversion)

**Design decision:** EasyOCR is lazy-loaded (expensive to initialize), so first OCR call may take 1–2 seconds for model download/initialization. Acceptable for one-time setup.
